HTML Semantics

First things first: before starting to follow this BRL Common Styles Guide, we advise you on trying to use HTML5’s semantics. Since this can be a nuanced subject - easily open to interpretation -, let's keep things simple: a semantic element clearly describes its meaning to both browser and developer. To not overthink your markup, consider:

  • Non-semantic elements like <div> and <span> tell nothing about its content, meaning they are generic block or inline containers usually used only for styling purposes.

  • Semantic elements, on the other hand, clearly define its content and allow you to organize the document content into logical pieces. Some examples of semantic elements are the well known <header>, <nav>, <main>, <section>, <article>, <p>, <footer>, among others.

Use these initial CSS references to organize your basic layout.

Usage

Use brl-container in main structures that can contain more than one element - e.g., a page wrapper and a sidebar.

The brl-page-wrapper should be inside a container and wraps around a specif content. Use it to constrain said content in the center of the page. This page wrapper uses media queries, which means it has a maximum width of 75% in a 1200px screen resolution, and the percentage enlarges as the screens get smaller.

Sizing

Width and height utilities cover sizing for 25%, 50%, 75%, and 100%.

Use brl-h or brl-w (for height or width), adding the sizing preference separated by a dash (-). You can also use brl-maxh or brl-maxw for maximum height/width (only for 100%). Examples: brl-w-100, for width of 100%; brl-w-75, for width of 75%, and from then on.

Spacing

These are spacing utilities that define margins and paddings. The classes are named using the format brl-{property}{sides}-{size}. Begin using brl-m- or brl-p- (for margin or padding).

Consider the sides as follow:

  • t - top;
  • b - bottom;
  • l - left;
  • r - right;
  • x - both *-left and *-right;
  • y - both *-top and *-bottom;
  • blank - for classes that set a margin or padding on all 4 sides of the element.

Consider the sizes:

  • 0 - to eliminate the margin or padding by setting it to 0;
  • 1
  • 2
  • 3
  • 4
  • 5
  • auto - for classes that set the margin to auto.

Usage

Margin and padding helpers consist of several classes ranging from 0 to 5, in which the values are set in rem units. Use them to set margin and padding throughout your layout. As examples, consider: brl-m-1, to set margin with values of 1 in all sides; or brl-pb-1, to set a value of 1 in the padding-bottom, and so.

Tip: Use brl-mx-auto to horizontally centralize a content in a box.

Spacing utilities also consider negative margins. Negative margins are named just like the regular margins, but with an extra dash (-). By using brl-mx--3, e.g., you would set a value of -3 on the x axis.

Flexbox

Usage

Flex Containers - Classes for the Parent elements

You can use brl-flex to create a flex container and enable a flex context for all its direct children. By default, it aligns the flex item(s) along the main axis, i.e., horizontally.

Use brl-hbox or brl-vbox to organize its children items along the main or cross axis:

  • Main axis - horizontally, from left to right: use brl-hbox;
  • Cross axis - vertically, from top to bottom: use brl-vbox;
Tip: Add the reverse class to lay out the flex items reversely in both directions. Row-reverse ( brl-hbox reverse) will align the items from right to left, and column-reverse ( brl-vbox reverse), from bottom to top.

Flex utilities can help you to make a container in which a main <div> with scrollable content has a fixed top and/or bottom bar (see Code & Examples). For that:

  • The outer <div> must be defined as a flex container with a vertical column - use brl-vbox;
  • The outer <div>'s maximum height must be set;
  • Use brl-main-content for the inner <div> with scrollable content;
  • You can define a top bar, a bottom bar, or both as options for the fixed bars.

Alerts

Alert messages must be used to provide contextual feedback for typical user actions, i.e., keep users informed of important and sometimes time-sensitive changes.

Usage

Start styling your alerts with brl-alert. You can add brl-alert-close and an optional dismiss button for dismissal alerts.

Following the initial styling, you can differentiate your alert messages between at least four types of messages:

  • Info messages (add brl-alert-info) let user know that something has happened in the system, but that this something is nothing out of the ordinary or unexpected. This type of message is not usually bound to a mandatory user interaction.

  • Success messages (add brl-alert-success) make it obvious to users that their interaction with the application was successful.

  • Warning messages (add brl-alert-warning) should appear when users are about to do something that can result in major changes or when the result of an action is unexpected - but isn't an error.

  • Error messages (add brl-alert-error) are used to inform users that something went wrong and, preferably, help them to fix it.

Tip: Style your alert message according to its type and severity. Also, when the user is required to do something in response to an alert, let them know what they need to do and make that task as easy as possible. Think about how much context to provide with your message.

Code & Examples

// Success alert
<div class="brl-alert brl-alert-success">
    <strong>Success!</strong> The changes have been saved.
</div>
    
// Success alert w/ dismiss button
<div class="brl-alert brl-alert-success brl-alert-close">
    <strong>Success!</strong> The changes have been saved.
    <button type="button" class="icon-close">
    </button>
</div>
Info! You have 20 unread notifications.
Success! The changes have been saved.
Warning! Your monthly traffic is reaching limit.
Error! Please enter a valid ID.

Buttons

The .brl-btn- classes are designed to be used with the <button> element. Start styling your buttons according with the guidelines as follow.

Usage

Primary & Secondary Colors Action Buttons

Primary buttons identify the main action in a main set of buttons, and use a $primary-color as color variable. Color variables may change depending of the project. Start styling primary buttons by using brl-btn-primary.

Secondary buttons are the main buttons in a secondary set of buttons. They use a $secondary-color as color variable, and are set with the brl-btn-secondary class.

Tip: Setting your project colors to primary and secondary color variables allow you to use your color theming for UI elements.

All buttons contain specific styles for disabled and active status. To make buttons look inactive, add the disabled boolean attribute. Active buttons require you to add the active class.

Code & Examples

// Primary Button
<button type="button" class="brl-btn-primary">
    Primary Button
</button>
    
// Secondary Button
<button type="button" class="brl-btn-secondary">
    Secondary Button
</button>
    
// Disabled Button
<button type="button" class="brl-btn-primary" disabled>
    Disabled
</button>
    
// Active Button
<button type="button" class="brl-btn-primary active">
    Active
</button>

Outline Action Buttons

Outline buttons identify secondary actions and have a weaker visual weight. Both primary and secondary colors action buttons have their outlined version. To use them, add brl-btn-outline.

In a primary set of buttons, an outline button is secondary to its primary filled button. Thus, consider:

However, in cases when an outline button has its own secondary action, use an outlined secondary color action button.

<button type="button" class="brl-btn-primary brl-btn-outline">
    Outline Primary Button
</button>

Ghost Action Buttons

Ghost buttons also identify secondary actions. To use them, add brl-btn-ghost.

Labeled Icon Buttons

To add font icons to your buttons, add brl-btn-icon, followed by brl-icon-[insert icon name] - see the list of available icons.

Note that brl-btn-icon only works if added to primary/secondary buttons and their outlined versions. 

<button type="button" class="brl-btn-primary brl-btn-icon icon-save">
    Label
</button>

Alert Colors Buttons

Icon Buttons

To create primary color icon buttons without a label, add only brl-icon-[insert icon name] to your button tag.

For secondary color icon buttons, also include the secondary-color class. 

// Icon Button (Primary Color)
<button type="button" class="icon-edit-pencil">
</button>

// Icon Button (Secondary Color)
<button type="button" class="icon-edit-pencil secondary-color">
</button>

Block-level Buttons

A block-level button spans its width according to the parent element, and can be defined by this guideline's sizing utilities.

To that effect, add brl-w-{percentage} (covers sizing for 25%, 50%, 75%, and 100%) to your button.

Forms

When creating forms, it is recommendable to use a <form> element to wrap the fields - read more. Fields contain inputs with a matching label.

Usage

The forms are styled with the brl-form class, and they can contain several fields. Style each field using brl-field.

You can use the <fieldset> element to create groups of fields that share the same purpose (e.g., insert several fields regarding user personal information, like name, e-mail etc.), for styling and semantic purposes. Using <div> works as well, although not semantically.

Tip: If you are planning on styling your fields with Flexbox, consider using a <div> as a flex container inside the <fieldset> element. Fieldset tags by themselves do not function with display: flex or inherited properties.

Here are some basic instructions on how to create your form (see Code & Examples, as follow):

  • Style your <form> or outer <div> with brl-form;
  • Insert a <fieldset> element (optional);
  • Style your field(s) with brl-field and consider:
    • If you wish to place more than one field per line, use Flexbox: style a div with brl-hbox and desirable margins and place your field inside it;
    • Distribute your fields inside the flex container by adding one of the following classes to each field: brl-flex-1, brl-flex-2, brl-flex-3, and so on;
    • For each field, insert an input and a matching label. Wrap the control form (input) inside the <label> element.

Code & Examples

// Form for one large field and one small
<form class="brl-form">
    <fieldset>
        <div class="brl-hbox brl-mx--2">
            <div class="brl-field brl-flex-2 brl-mx-2">
                <label>Large Field:</label>
                <input class="brl-input-default" type="text">
            </div>
            <div class="brl-field brl-flex-2 brl-mx-2">
                <label>Small Field:</label>
                <input class="brl-input-default" type="text">
            </div>
        </div>
    </fieldset>
</form>
